  <files>
    lighthouserc.js
    package.json
  </files>
  <action>
    1. Install @lhci/cli as a devDependency:
       ```bash
       npm install -D @lhci/cli
       ```

    2. Create lighthouserc.js in the project root with this configuration:
       ```javascript
       module.exports = {
         ci: {
           collect: {
             staticDistDir: './dist',
             numberOfRuns: 3,
             url: [
               'http://localhost/index.html',
               'http://localhost/pricing/index.html',
               'http://localhost/why-fierro/index.html',
               'http://localhost/privacy/index.html',
               'http://localhost/terms/index.html',
             ],
           },
           assert: {
             assertions: {
               'categories:performance': ['error', { minScore: 0.95 }],
               'categories:accessibility': ['error', { minScore: 0.95 }],
               'categories:best-practices': ['error', { minScore: 0.95 }],
             },
           },
           upload: {
             target: 'filesystem',
             outputDir: '.lighthouseci',
           },
         },
       };
       ```

       Key details per research:
       - numberOfRuns: 3 takes median to reduce flakiness
       - staticDistDir: lhci spins up its own server, no Astro dev server needed
       - Explicit URL list because lhci auto-detection misses nested dirs (known limitation)
       - upload to filesystem -- no external LHCI server

    3. Add npm scripts to package.json:
       ```json
       "test:lighthouse": "lhci autorun",
       "test:all": "npm run build && npm run test:lighthouse && npm run test"
       ```

       These go alongside the existing "test" and "test:chromium" scripts.

    4. Verify the installation and config work:
       ```bash
       npm run build && npx lhci autorun
       ```

       If any Lighthouse score is below 0.95, investigate and fix the issue before proceeding. Common fixes per research:
       - If accessibility score is low, check for missing alt text, contrast issues, or landmark problems (should be fixed by Plan 01)
       - If performance is low, check for unoptimized images flagged by Lighthouse
       - The ClientRouter JS (16KB / 5.3KB gzip) is expected and acceptable -- do NOT remove it
       - If image optimization issues are flagged, fix only what Lighthouse specifically identifies (per user decision: "Image optimization is Lighthouse-driven only")

       IMPORTANT: If Lighthouse flags specific issues that require code changes to hit 95+, fix them in this task. The user decision is strict: "Strict 95+ on Lighthouse Performance, Accessibility, and Best Practices for every page -- no exceptions." Fix what needs fixing.
