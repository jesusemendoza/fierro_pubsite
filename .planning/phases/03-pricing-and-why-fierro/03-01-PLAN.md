**PricingToggle.astro** -- Vanilla JS custom element for monthly/annual toggle:
- Wrap content in `<pricing-toggle data-default="monthly">` custom element tag
- Toggle UI: flex centered row with "Monthly" label (font-semibold by default), toggle button (`role="switch"`, `aria-checked="false"`, `aria-label="Switch to annual billing"`), "Annual" label (text-concrete-gray by default)
- Toggle button: relative h-7 w-14 rounded-full bg-concrete-gray/30, with absolute-positioned span thumb (h-5 w-5 rounded-full bg-white left-1 top-1, translate-x-7 when active)
- Savings callout: `<p data-savings>` with "Save $118/year" in rebar-green, hidden by default, shown when annual is selected
- `<script>` tag (NOT `<script is:inline>`) defining `class PricingToggle extends HTMLElement` with `connectedCallback()`:
  - On button click: toggle `isAnnual` boolean
  - Update `aria-checked`, `aria-label` (switch between "Switch to monthly/annual billing")
  - Toggle thumb position (translate-x-7), button bg (bg-molten-orange when annual, bg-concrete-gray/30 when monthly)
  - Toggle label styles (font-semibold on active, text-concrete-gray on inactive)
  - Show/hide savings callout
  - Query ALL `[data-price-monthly]` elements on the page, swap textContent between monthly/annual values
  - Query ALL `[data-billing-link]` elements, update href billing param (replace `billing=monthly` with `billing=annual` and vice versa)
  - Show/hide `[data-annual-note]` elements (visible when annual, hidden when monthly)
- Register: `customElements.define('pricing-toggle', PricingToggle)` -- connectedCallback fires automatically on view transitions, no astro:page-load listener needed
