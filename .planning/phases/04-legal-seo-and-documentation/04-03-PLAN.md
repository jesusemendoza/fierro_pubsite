2. **Write ../fierro_web/docs/plans/deep-link-signup-flow.md** covering:
   - **Overview:** The marketing site (getfierro.com) pricing page uses deep-linked CTAs that pass plan context to the app's signup flow via query parameters. This doc describes the contract so fierro_web can implement param reading and auto-routing.
   - **Query parameter contract:**
     - `plan` parameter: `"free"` | `"plus"` | `"builder"` (optional -- absence means default signup)
     - `billing` parameter: `"monthly"` | `"annual"` (optional -- only relevant for Plus tier)
     - Base URL: `https://app.getfierro.com/signup`
   - **URL examples from marketing site:**
     - Free tier CTA: `https://app.getfierro.com/signup` (no params)
     - Plus Monthly: `https://app.getfierro.com/signup?plan=plus&billing=monthly`
     - Plus Annual: `https://app.getfierro.com/signup?plan=plus&billing=annual`
     - Builder: `mailto:sales@getfierro.com` (no signup flow -- contact sales)
   - **Implementation guidance for fierro_web:**
     - On signup page load, read `plan` and `billing` from `searchParams`
     - If `plan=plus`: after successful account creation, auto-redirect to Stripe Checkout for the Plus plan
     - If `billing=annual`: pre-select annual billing in Stripe Checkout (use annual price ID)
     - If `billing=monthly` or absent: default to monthly billing
     - If no `plan` param: standard free signup flow (no Stripe redirect)
     - If `plan=free`: same as no plan param -- standard free signup
     - Store plan intent in session/state during signup so it survives the auth flow (Supabase email verification, OAuth redirect, etc.)
   - **Stripe integration points:**
     - Reference stripe-billing-plan.md for Stripe price IDs and checkout session creation
     - After account creation + email verification, redirect to `/api/stripe/checkout?plan={plan}&billing={billing}` (or equivalent endpoint)
     - Checkout success URL should land on the app dashboard
   - **MCP server integration points:**
     - MCP server access is gated by plan tier (Plus and Builder only)
     - After signup with plan=plus, the user's account should have MCP access enabled
     - Reference the MCP server configuration in fierro_web for connection details
   - **Edge cases:**
     - Invalid plan value: ignore, treat as free signup
     - Invalid billing value: ignore, default to monthly
     - User already logged in: redirect to dashboard or billing upgrade page
     - Stripe Checkout cancellation: land user on app dashboard with free plan active

   Audience: developer working on fierro_web -- clear contract, specific param names, actionable implementation steps.
