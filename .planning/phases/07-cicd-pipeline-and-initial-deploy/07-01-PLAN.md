  <name>Task 1: Create GitHub Actions workflow and update Playwright CI config</name>
  <files>.github/workflows/deploy.yml, playwright.config.ts</files>
  <action>
**1. Create `.github/workflows/deploy.yml`** with a single-job sequential pipeline:

- **Trigger:** `on: push: branches: [main]` only. No PR triggers, no other branches.
- **Job:** `deploy` on `ubuntu-latest` with `timeout-minutes: 15`.
- **Steps (in order):**
  1. `actions/checkout@v4`
  2. `actions/setup-node@v4` with `node-version: 22` and `cache: npm`
  3. `npm ci` (Install dependencies)
  4. `npm run build` (Build site)
  5. `npx playwright install chromium --with-deps` (Install Playwright browsers — chromium only, with OS deps)
  6. `npx playwright test` (Run tests — Playwright will auto-start preview server via config)
  7. `actions/upload-artifact@v4` with `if: ${{ !cancelled() }}` to upload `playwright-report/` with `retention-days: 7` (uploads on failure too)
  8. `cloudflare/wrangler-action@v3` with `apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}`, `accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}`, `command: deploy`
  9. Post-deploy smoke check: curl retry loop (5 attempts, 10s between, `--max-time 10`) that checks `https://getfierro.com` returns HTTP 200. Fail the step if all 5 attempts fail.

**2. Update `playwright.config.ts`** — change the `webServer.command` to be CI-aware:

```typescript
webServer: {
  command: process.env.CI ? 'npm run preview' : 'npm run dev',
  url: 'http://localhost:4321',
  reuseExistingServer: !process.env.CI,
  timeout: 30000,
},
```

This ensures CI runs tests against the built `dist/` folder (production fidelity) while local dev continues using the hot-reload dev server. Port 4321 is the same for both `astro dev` and `astro preview`, so `baseURL` does not change.

**Do NOT:**
- Add Lighthouse CI to the workflow (deferred per user decision)
- Add PR triggers or branch preview deploys (deferred per user decision)
- Cache Playwright browsers (Playwright docs recommend against it — restore time equals download time)
- Split into multiple jobs (unnecessary for a simple static site pipeline)
- Add `account_id` to `wrangler.jsonc` (keep it as a CI secret for flexibility)

**2. CI/CD Setup** (new subsection)
Step-by-step instructions for first-time setup:

a. **Create Cloudflare API Token:**
   1. Go to https://dash.cloudflare.com/profile/api-tokens
   2. Click "Create Token"
   3. Find "Edit Cloudflare Workers" template, click "Use template"
   4. Under Account Resources: select "Include" > your account name
   5. Under Zone Resources: select "Include" > "All Zones" (or specific zone for getfierro.com)
   6. Click "Continue to summary" > "Create Token"
   7. Copy the token immediately (shown only once)

b. **Find Cloudflare Account ID:**
   - Visible in the Cloudflare dashboard URL: `https://dash.cloudflare.com/<account-id>`
   - Also shown in the right sidebar of the Workers & Pages overview

c. **Add GitHub Repository Secrets:**
   1. Go to GitHub repo > Settings > Secrets and variables > Actions
   2. Click "New repository secret"
   3. Add `CLOUDFLARE_API_TOKEN` with the token value
   4. Add `CLOUDFLARE_ACCOUNT_ID` with the account ID value

**3. DNS & Custom Domain** (new subsection)
- Cloudflare DNS must have `getfierro.com` proxied (orange cloud icon)
- `wrangler.jsonc` has `custom_domain: true` route — Cloudflare handles SSL automatically after first `wrangler deploy`
- To verify: `curl -I https://getfierro.com` should show `cf-ray` header and valid SSL
- Note: first deploy with custom domain may take a few minutes for DNS propagation

**4. Troubleshooting** (new subsection)
Short FAQ format covering:

| Problem | Cause | Fix |
|---------|-------|-----|
| Deploy fails with 403 | API token wrong permissions or expired | Recreate token using "Edit Cloudflare Workers" template |
| Deploy fails with account routing error | Missing `CLOUDFLARE_ACCOUNT_ID` secret | Add account ID as GitHub secret |
| Build fails in CI but works locally | Node version mismatch or missing dependency | Check `node-version: 22` in workflow, run `npm ci` (not `npm install`) |
| Tests fail in CI but pass locally | Dev vs preview server difference, or flaky test | CI uses `npm run preview` (production build); check test assumes dev-only behavior |
| Smoke check fails after deploy | Edge propagation delay (normally < 30s) | Re-run workflow; if persistent, check Cloudflare Workers dashboard for errors |
| Domain not routing to site | DNS not proxied or custom domain not configured | Verify orange cloud in Cloudflare DNS, run `wrangler deploy` to register route |

**Audience:** "Future me" — concise commands, covers gotchas, no hand-holding on basics (per Phase 4 context decision).
